# Codex Project Prompt: HD현대일렉트릭 수치형 데이터 기반 ML + Local LLM Serving Program

당신은 이 저장소를 정리하고 구현하는 Codex 엔지니어다. 목표는 기존 `HD_LLM.ipynb`의 분류/회귀 ML 로직을 재현 가능한 Python 패키지로 분리하고, Streamlit GUI에서 사용자가 Excel 데이터를 업로드하여 학습/검증/예측/LLM 질의응답을 수행할 수 있는 local LLM 기반 serving 프로그램을 만드는 것이다.

이 프롬프트의 요구사항을 우선순위 높게 따른다. 구현 중 불명확한 부분이 있으면 합리적인 MVP 결정을 내리고, README에 결정 내용을 명시한다.

## 0. 현재 업로드/소스 파일 역할

현재 프로젝트의 입력 파일은 다음과 같다.

- `HD_LLM.ipynb`: 핵심 notebook 코드. PyCaret 기반 classification/regression 모델 탐색 코드가 들어 있다.
- `class_extracted.xlsx`: classification용 수치형 Excel 데이터.
- `Reg_extracted.xlsx`: regression용 수치형 Excel 데이터.
- `HD현대일렉트릭_과제중간발표.pptx`: 프로젝트 개요, 문제 정의, 성능 해석, SHAP/이미지 DB 계획 등이 담긴 발표자료.
- `파일-설명.txt`: 파일 역할 설명. 엑셀은 데이터, PPT는 프로젝트 개요, ipynb는 핵심 코드라고 설명한다.

발표자료 기준 프로젝트 목표는 형상/시험 변수 기반 고압차단기 성능 예측 모델을 만드는 것이다. 수치형 DB로 회귀와 분류 작업을 수행했고, 회귀가 어렵기 때문에 차단 성공/실패 이진분류 문제로 전환하는 방향이 포함되어 있다. 분류 target은 `Result` 열을 사용하며, `Result == 1.0`이면 차단성공 Class 1, 그 외는 차단실패 Class 0으로 정의한다.

## 1. 기존 notebook에서 반드시 반영할 로직

### 1.1 Classification notebook logic

기존 notebook의 classification 로직은 다음과 같다.

```python
import numpy as np
import pandas as pd
from pycaret.classification import *

df = pd.read_excel('C:/Users/user/class_extracted.xlsx')
df = df.dropna()
df = df.reset_index(drop=True)

label = np.where(df['Result'] == 1.0, int(1), int(0))
label = label.reshape(-1, 1)

df['label'] = label
df = df.drop(['Result', 'TRVmax[kV]'], axis=1)

setup = setup(data=df, target='label', session_id=0, train_size=0.9)
best_model = compare_models()
evaluate_model(best_model)
```

반드시 유지할 핵심 정의:

- classification label: `Result == 1.0 -> 1`, else `0`.
- classification target column: 새로 만든 `label`.
- classification에서 feature로 쓰면 안 되는 column: `Result`, `TRVmax[kV]`.
- notebook의 절대경로 `C:/Users/user/...`는 절대 사용하지 말고 project-relative path 또는 Streamlit uploaded file을 사용한다.
- notebook처럼 `dropna()`를 수행하되, 제거된 row 수를 UI와 logs에 표시한다.

### 1.2 Regression notebook logic

기존 notebook의 regression 로직은 다음과 같다.

```python
import numpy as np
import pandas as pd
from pycaret.regression import *

df_all = pd.read_excel('C:/Users/user/Reg_extracted.xlsx')
df_all = df_all.dropna()
df_all = df_all.reset_index(drop=True)

df_all = df_all.drop(['Time', 'Result', 'CZM', 'Test'], axis=1)
first_col = df_all.columns[0]
df_all = df_all[df_all.columns[1:].tolist() + [first_col]]

setup = setup(data=df_all, target='TRVmax[kV]', session_id=0, train_size=0.9)
best_model = compare_models()
evaluate_model(best_model)
```

반드시 유지할 핵심 정의:

- regression target: `TRVmax[kV]`.
- regression에서 feature로 쓰면 안 되는 column: `Time`, `Result`, `CZM`, `Test`, `TRVmax[kV]`.
- notebook에서는 `TRVmax[kV]`를 마지막 column으로 옮긴 뒤 target으로 사용한다. 구현에서는 column 순서를 schema로 명확히 관리하면 된다.
- notebook의 절대경로는 사용하지 않는다.

## 2. 최종 목표

다음 기능을 갖는 Streamlit app과 Python source package를 구현한다.

1. 사용자가 Streamlit GUI에서 `.xlsx` 또는 `.xls` Excel 파일을 업로드할 수 있다.
2. 업로드된 Excel을 pandas로 읽고, 데이터 미리보기, shape, column 목록, 결측치 summary, target 분포를 보여준다.
3. 업로드 데이터가 다음 중 무엇인지 자동 추정하고, 사용자가 수동으로 변경할 수 있다.
   - `classification_training`
   - `regression_training`
   - `classification_prediction`
   - `regression_prediction`
   - `unknown`
4. 사용자는 업로드된 training data로 classification/regression 모델을 학습하고 artifact를 저장할 수 있다.
5. 사용자는 저장된 model artifact로 단건 또는 batch Excel prediction을 수행할 수 있다.
6. batch prediction 결과를 화면에 보여주고 Excel/CSV로 다운로드할 수 있다.
7. local LLM chat UI를 제공한다.
8. LLM은 tool calling을 통해서만 데이터 요약, schema 검증, 학습, 예측, 결과 설명을 수행한다.
9. LLM은 예측값을 임의 생성하거나 hallucination하면 안 된다. 예측은 반드시 deterministic ML tool 결과를 기반으로 한다.
10. Streamlit UI는 저장된 model artifact를 로딩하여 inference를 수행한다. 앱 rerun마다 model training을 반복하지 않는다.

## 3. 기술 스택

필수:

- Python 3.10+
- Streamlit
- pandas
- numpy
- scikit-learn
- joblib
- openpyxl
- requests
- python-dotenv

권장/선택:

- shap: 가능하면 local/global explanation에 사용. 설치되지 않았거나 실패하면 fallback 사용.
- imbalanced-learn: classification에서 BorderlineSMOTE 옵션 제공. 설치되지 않았으면 해당 옵션을 비활성화하고 명확히 표시.
- PyCaret: 기존 notebook parity용 선택 dependency. 단, Streamlit runtime이 PyCaret에 강하게 의존하면 안 된다.

원칙:

- MVP serving은 가벼운 scikit-learn/joblib artifact 기반으로 동작해야 한다.
- 기존 notebook의 PyCaret `compare_models()`는 참고용이며, production Streamlit app 안에서 직접 실행하지 않는다.
- PyCaret trainer를 추가하고 싶다면 `src/hd_serving/pycaret_train.py`처럼 별도 optional module로 둔다.

## 4. Local LLM serving 구조

기본 provider는 Ollama로 한다.

환경변수:

```bash
LLM_PROVIDER=ollama
OLLAMA_BASE_URL=http://localhost:11434
OLLAMA_MODEL=gemma4:e2b
LLM_TIMEOUT_SECONDS=60
```

요구사항:

- `src/hd_serving/llm_client.py`에 provider adapter를 만든다.
- 기본 adapter는 Ollama `/api/chat`을 사용한다.
- Ollama tool calling을 지원하는 모델이면 `tools` parameter를 사용하여 function/tool calling을 수행한다.
- tool calling 미지원 모델이거나 tool call parse에 실패하면, 안전한 fallback router를 사용한다. fallback은 명시적 키워드 기반으로만 동작하고 예측값을 임의 생성하지 않는다.
- 향후 vLLM/OpenAI-compatible endpoint로 교체할 수 있도록 `openai_compatible` adapter skeleton을 둔다.

LLM의 역할:

- 프로젝트/코드/데이터/모델 설명.
- 사용자의 자연어 요청을 tool call로 변환.
- tool 결과를 읽고 설계자 친화적인 한국어 설명을 생성.
- 누락 column, schema mismatch, model artifact 없음 같은 문제를 명확히 사용자에게 설명.

LLM 금지사항:

- 업로드된 Excel 전체를 prompt에 그대로 넣지 않는다.
- 예측값, 확률, metric, SHAP 값을 임의로 말하지 않는다.
- schema에 없는 feature를 상상해서 채우지 않는다.
- 누락 feature를 평균값 등으로 자동 대체하지 않는다. 단, 사용자가 명시적으로 결측치 처리 옵션을 선택한 경우에만 허용한다.
- Python `eval`, shell command execution 등 위험한 tool을 제공하지 않는다.

## 5. Tool calling 설계

`src/hd_serving/tools.py`에 LLM이 호출할 수 있는 deterministic tools를 구현한다. 각 tool은 JSON-serializable input/output을 갖고, 실패 시 structured error를 반환한다.

필수 tools:

1. `get_uploaded_data_summary(dataset_id: str | None = None) -> dict`
   - active uploaded dataframe의 row/column 수, column list, missing summary, target 분포를 반환.
   - dataframe 전체 원문을 반환하지 말고 sample preview는 최대 5행만 반환.

2. `infer_uploaded_dataset_type(dataset_id: str | None = None) -> dict`
   - column set 기반으로 데이터 유형을 추정.

3. `validate_classification_schema(dataset_id: str | None = None, model_id: str = "latest") -> dict`
   - 저장된 classification schema와 업로드 데이터 column을 비교.
   - missing columns, extra columns, ignored columns를 반환.

4. `validate_regression_schema(dataset_id: str | None = None, model_id: str = "latest") -> dict`
   - 저장된 regression schema와 업로드 데이터 column을 비교.

5. `train_classification_model(dataset_id: str | None = None, options: dict | None = None) -> dict`
   - classification training pipeline 실행.
   - artifact 저장 후 metrics/schema/model_card path를 반환.

6. `train_regression_model(dataset_id: str | None = None, options: dict | None = None) -> dict`
   - regression training pipeline 실행.

7. `predict_classification_batch(dataset_id: str | None = None, model_id: str = "latest") -> dict`
   - batch prediction 수행.
   - 결과 dataframe은 session registry에 저장하고, summary와 result_id를 반환.
   - classification output: `prediction`, `prediction_label`, `probability_success`, `probability_failure`.

8. `predict_regression_batch(dataset_id: str | None = None, model_id: str = "latest") -> dict`
   - regression batch prediction 수행.
   - output: `predicted_TRVmax[kV]`.

9. `explain_model_or_prediction(task: str, model_id: str = "latest", dataset_id: str | None = None, row_index: int | None = None) -> dict`
   - SHAP 사용 가능 시 SHAP summary/local explanation.
   - 실패 시 built-in feature_importances_, coef_, permutation importance 순서로 fallback.

10. `get_model_metrics(task: str, model_id: str = "latest") -> dict`
    - `metrics.json`, `model_card.md` 기반 summary 반환.

11. `get_prediction_result_summary(result_id: str) -> dict`
    - batch prediction 결과 요약 반환.

도구 출력은 LLM이 그대로 읽을 수 있게 한국어 label과 machine-readable key를 함께 둔다.

## 7. 데이터 판별 로직

`src/hd_serving/schema.py` 또는 `preprocessing.py`에 구현한다.

```python
def infer_dataset_type(columns: list[str]) -> str:
    colset = set(columns)
    has_result = "Result" in colset
    has_trv = "TRVmax[kV]" in colset
    has_reg_meta = {"Time", "CZM", "Test"}.issubset(colset)

    if has_result and has_trv and has_reg_meta:
        return "regression_training"
    if has_result and has_trv:
        return "classification_training"
    return "prediction_input_or_unknown"
```

추론 후 사용자가 manual override할 수 있어야 한다.

## 8. Preprocessing 세부사항

### 8.1 Classification preprocessing

```python
def prepare_classification_training_df(df):
    original_rows = len(df)
    df = df.dropna().reset_index(drop=True)
    dropped_rows = original_rows - len(df)

    y = np.where(df["Result"] == 1.0, 1, 0)
    X = df.drop(columns=["Result", "TRVmax[kV]"], errors="raise")
    return X, y, {"dropped_rows": dropped_rows}
```

주의:

- `label`을 feature에 포함시키면 안 된다.
- `Result`, `TRVmax[kV]`는 classification feature에 포함시키면 안 된다.
- `errors="raise"`로 필수 column 누락을 조기에 잡는다.

### 8.2 Regression preprocessing

```python
def prepare_regression_training_df(df):
    original_rows = len(df)
    df = df.dropna().reset_index(drop=True)
    dropped_rows = original_rows - len(df)

    y = df["TRVmax[kV]"]
    X = df.drop(columns=["Time", "Result", "CZM", "Test", "TRVmax[kV]"], errors="raise")
    return X, y, {"dropped_rows": dropped_rows}
```

주의:

- `TRVmax[kV]`는 target이며 feature에 포함시키면 안 된다.
- `Time`, `Result`, `CZM`, `Test`는 regression feature에 포함시키면 안 된다.

### 8.3 Prediction preprocessing

- 저장된 `schema.json`의 `features` 목록을 기준으로 column을 정렬한다.
- 업로드 dataframe에 target/ignored columns가 있으면 prediction 시에는 무시할 수 있다. 단, ignored columns 목록을 사용자에게 보여준다.
- missing feature가 있으면 예측하지 않는다.
- extra columns는 무시 가능하지만, 무시된 column 목록을 결과에 포함한다.

## 9. Schema artifact 형식

## 11. Explanation 요구사항

`src/hd_serving/explanation.py` 구현.

우선순위:

1. SHAP 사용 가능하면 SHAP 계산.
2. 모델에 `feature_importances_`가 있으면 이를 사용.
3. 선형 모델에 `coef_`가 있으면 이를 사용.
4. 위 모두 불가능하면 permutation importance 사용.
5. permutation importance도 실패하면 explanation unavailable error를 반환.

결과는 다음 구조로 반환한다.

```json
{
  "task": "classification",
  "method": "feature_importances_",
  "top_features": [
    {"feature": "MN_L1", "importance": 0.18},
    {"feature": "MN_S2", "importance": 0.05}
  ],
  "notes": "SHAP unavailable; used model.feature_importances_ fallback."
}
```

중요:

- explanation 값도 임의 생성하지 않는다.
- SHAP 계산 실패는 정상적인 fallback 대상이다.

## 18. 보안/안전 기준

- 사용자가 업로드한 Excel 파일을 코드로 실행하지 않는다.
- LLM output을 Python 코드로 실행하지 않는다.
- Shell command를 LLM tool로 제공하지 않는다.
- API key가 필요한 경우 `.env`로만 읽고 코드에 하드코딩하지 않는다.
- 로컬 LLM이 실패하면 cloud fallback을 자동 사용하지 않는다.
- 사용자의 업로드 데이터를 외부 API로 전송하지 않는다.

## 19. 구현 시 주의할 불일치 가능성

PPT에 기록된 과거 성능값과 현재 업로드 Excel/notebook output이 다를 수 있다. 따라서 다음 원칙을 지킨다.

- PPT 성능값을 app에 하드코딩하지 않는다.
- 현재 업로드된 Excel과 현재 학습 pipeline으로 계산한 metric을 `metrics.json`에 저장한다.
- UI에서는 저장된 `metrics.json` 값을 보여준다.
- README에는 “데이터 버전과 preprocessing에 따라 성능이 달라질 수 있다”고 명시한다.

## 22. 가장 중요한 원칙

이 프로젝트의 핵심은 “LLM이 예측 모델처럼 말하는 것”이 아니라, “LLM이 deterministic ML tools를 호출하고 그 결과를 설명하는 것”이다.

따라서 다음 원칙을 절대 어기지 않는다.

- ML 예측은 ML artifact가 한다.
- LLM은 tool call orchestration과 설명을 담당한다.
- Excel 데이터는 Streamlit upload/session registry로 관리한다.
- schema 검증 없이 prediction하지 않는다.
- app runtime에서 notebook을 직접 실행하지 않는다.
- 성능값을 하드코딩하지 않는다.
- 절대경로를 사용하지 않는다.
